Q 这个软件适合我吗？

本软件主要面向以下用户：

1. 需要在特定区域宣发的创作者/社团 — 比如你想把高校例会信息精准推送到本地的东方群。

2. 需要频繁进行群发消息的搬运工 — 软件可以批量勾选群组，提高转发效率。

3. 自己加了大量东方群且有宣发需求的用户 — 软件内置 1800+ 群列表只是全集，加的群越多，宣发覆盖面越广。

如果您不属于以上用户，但仍想为建设便捷的宣发软件尽自己的一份力量，同样欢迎试用软件或进群讨论！

Q 这个软件安全吗？会上传我的个人信息吗？

绝对不会。原初电台不会、也无法上传你的任何个人信息。

软件上传到服务器的内容仅有经过用户同意的群列表增量信息，用途仅限于帮助所有用户同步完善群列表，除此之外任何软件运行过程中产生的数据均在本地完成处理。

Q 启动报"未找到 QQ"但我明明装了 QQ

原初电台基于 NapCat 框架开发，该框架需要 Electron 架构的新版 QQ，旧版 Win32 QQ 不支持。

下载前先自查：打开 QQ，点击左下角菜单 → 关于，查看版本号。目前已验证 9.920 ~ 9.926 均可正常使用。

如果版本太旧（9.9.20 以下）或太新（超出 NapCat 兼容范围），请统一安装推荐版本：

QQ9.9.26.44343_x64.exe

安装后建议关闭 QQ 自动更新：打开 QQ 设置 → 软件更新，取消勾选"有更新时自动安装"。这样就不会因为自动升级导致版本超出兼容范围。

软件会自动扫描 C/D/E 等所有盘符下的 QQ 安装位置，无需手动指定路径。

Q 扫码登录正常但提示"等待超时"

NapCat 已启动并显示二维码，但软件连不上它的 HTTP 服务（5700 端口）。

最常见原因：

1. QQ 进程残留 — 上次崩溃的 QQ 进程没杀干净，导致 NapCat 注入失败。可以复制以下命令到 PowerShell 中运行：

taskkill /F /IM QQ.exe; taskkill /F /IM NapCatWinBootMain.exe

或打开任务管理器手动结束QQ和NapCat进程。

2. Windows Defender 防火墙弹窗 — 第一次运行时 Windows 可能会弹框问"是否允许访问网络"。不过通常软件和 QQ 同在本机回环通信，一般不触发防火墙。如果确实遇到弹窗，请点"允许"。

Q 为什么启动软件后我的 QQ 自动退出了？

这是正常的！NapCat 需要接管你的 QQ 进程来实现 Bot 功能。启动时会先关闭你当前的 QQ，然后以注入模式重新拉起。

这意味着启动软件期间你不能同时使用这个 QQ 号。关闭软件后即可重新正常登录 QQ。

Q 为什么页面上说 1800+ 群，我的群列表里只有几十个？

群列表里显示的是两个来源的交集：

1. 内置 CSV — 从 THBWiki 爬取的约 1800 个东方群

2. 你 Bot 账号实际加入的群 — 载入群列表时会调用 QQ API 获取你加入的群

显然软件做不到向你不在的群里发送消息！如果你看到只有几十个，说明当前你加入的记录在 THBWiki 中的东方群只有几十个，试试加入更多的东方群来提升宣发范围，或等待后续的共享账号更新！

Q 可以手动添加群聊吗？

可以！本软件支持手动添加群聊功能（菜单 → 群列表 → 添加群聊）。填写群号和分类信息后即可加入本地列表。

提交时选择"添加到云端"，你的群聊信息会提交给维护者审核，审核通过后所有本软件的用户都能同步到。东方人帮帮东方人，一起维护这份群列表 🌸

注意：当前群列表数据来源于 THBWiki 自动爬取，分类体系并不完全匹配本软件的使用场景。后续有时间将彻底重新整理一版分类，敬请期待。
