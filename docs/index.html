常见问题

安装和使用中可能遇到的坑

Q 启动报"未找到 QQ"但我明明装了 QQ

宣发姬需要 Electron 架构的新版 QQ，旧版 Win32 QQ 不支持。

下载前先自查：打开 QQ，点击左下角菜单 → 关于，查看版本号。目前已验证 9.920 ~ 9.926 均可正常使用。

如果版本太旧（9.9.20 以下）或太新（超出 NapCat 兼容范围），请统一安装推荐版本：

QQ9.9.26.44343_x64.exe

安装后建议关闭 QQ 自动更新：打开 QQ 设置 → 软件更新，取消勾选"有更新时自动安装"。这样就不会因为自动升级导致版本超出兼容范围。

宣发姬会自动扫描 C/D/E 等所有盘符下的 QQ 安装位置，无需手动指定路径。

Q 扫码登录正常但提示"等待超时"

NapCat 已启动并显示二维码，但宣发姬连不上它的 HTTP 服务（5700 端口）。

最常见原因：

1. QQ 进程残留 — 上次崩溃的 QQ 进程没杀干净，导致 NapCat 注入失败。可以复制以下命令到 PowerShell 中运行：

taskkill /F /IM QQ.exe; taskkill /F /IM NapCatWinBootMain.exe

2. Windows Defender 防火墙弹窗 — 第一次运行时 Windows 可能会弹框问"是否允许访问网络"。不过通常宣发姬和 QQ 同在本机回环通信，一般不触发防火墙。如果确实遇到弹窗，请点"允许"。

Q 快速登录点完没有反应

快速登录需要之前成功登录过一次，并且 QQ 服务器端保留了会话。如果换过设备、QQ 密码、或服务器端清除了会话，快登会失败。

此时 NapCat 会自动降级到二维码扫码模式，重新扫码登录一次即可恢复快登缓存。

Q 旧版 QQ 改的文件夹名含"QQNT"但实际不是

有人会把旧版 QQ 的文件夹改名来骗过检测。宣发姬会通过以下方式识别真正的 Electron 版 QQ：

1. 路径是否包含 "QQNT" 字样

2. 目录下是否有 QQNT.dll、wrapper.node、misc.bin 等标记文件

3. QQ.exe 的文件版本号 build ≥ 40000（Electron 版 QQ 的特有范围，旧版只有几百）

三条满足其一就算识别通过，不必担心误判。

Q 可以手动添加群聊吗？

可以！宣发姬支持手动添加群聊功能（菜单 → 群列表 → 添加群聊）。填写群号和分类信息后即可加入本地列表。

提交时选择"添加到云端"，你的群聊信息会提交给维护者审核，审核通过后所有宣发姬用户都能同步到。东方人多帮帮东方人，一起维护这份群列表 🌸

注意：当前群列表数据来源于 THBWiki 自动爬取，分类体系并不完全匹配宣发姬的使用场景。后续会找时间彻底重新整理一版分类，敬请期待。

Q 群列表会自动更新吗？

会的。宣发姬启动时会自动连接服务器，增量同步最新的群列表变更（类 Git 维护模式），只拉取有变化的部分，不重复下载完整 CSV。

同时软件本身也内置了自动更新检查，在帮助菜单中也可以手动"检查更新"。有新版本时会在启动时提示，一键下载升级。
