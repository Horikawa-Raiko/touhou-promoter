常见问题

安装和使用中可能遇到的坑

Q 启动报"未找到 QQNT"但我明明装了 QQ

你装的可能是旧版 Win32 QQ，NapCat 只支持 QQNT（Electron 架构的新版 QQ）。

到 im.qq.com 下载"全新架构版"，安装后再试。

如果已确认是 QQNT 仍然报错：你可能是从 D 盘迁移或卸载重装过，注册表留下了旧路径但实际文件已不存在。在设置里手动指定 QQNT 安装目录即可。

Q QQ 版本太新，Worker 反复崩溃

QQNT 自动更新推的版本可能超出 NapCat 的兼容范围。

安装 NapCat 官方推荐版本：QQ9.9.26.44343_x64.exe

下载地址：https://dldir1.qq.com/qqfile/qq/QQNT/...

安装后务必阻止 QQNT 自动更新：到 QQNT 安装目录下，把 updater.exe 改名（如 updater.exe.bak），否则过两天又被自动升级回去。

Q QQNT 版本太旧被拒绝

QQNT build 号过低（一般从旧安装包或第三方下载站获取）。

到 这里 下载官方推荐版本重新安装。

Q QQ 装在 D 盘或自定义路径找不到

QQNT 装在非标准路径（如 D:\QQ\、E:\D盘迁移\Program Files(x86)\Tencent\QQNT\）时可能搜不到。

在设置中手动指定 QQNT 安装目录即可。指定目录就行，不用精确到 QQ.exe，程序会自动扫描版本子目录。

Q 扫码登录正常但提示"等待超时"

NapCat 已启动并显示二维码，但宣发姬连不上它的 HTTP 服务（5700 端口）。

最常见原因：

1. Windows 防火墙弹窗 — 第一次运行时 Windows 会弹框问"是否允许访问网络"，如果点了取消或被挡在后面，后续请求全部超时。检查任务栏或 Alt+Tab 有没有隐藏的防火墙弹窗。

2. QQ 进程残留 — 上次崩溃的 QQ 进程没杀干净，导致 NapCat 注入失败。打开任务管理器杀掉所有 QQ.exe 和 NapCatWinBootMain.exe 进程后重试。

Q 快速登录点完没有反应

快速登录需要之前成功登录过一次，并且 QQ 服务器端保留了会话。如果换过设备、QQ 密码、或服务器端清除了会话，快登会失败。

此时 NapCat 会自动降级到二维码扫码模式，重新扫码登录一次即可恢复快登缓存。

Q 旧版 QQ 改的文件夹名含"QQNT"但实际不是

有人会把旧版 QQ 的文件夹改名来骗过检测。宣发姬会通过以下方式识别真正的 QQNT：

1. 路径是否包含 "QQNT" 字样

2. 目录下是否有 QQNT.dll、wrapper.node、misc.bin 等标记文件

3. QQ.exe 的文件版本号 build ≥ 40000（QQNT 的特有范围，旧版 QQ 只有几百）

三条满足其一就算 QQNT，不必担心误判。
